This palette is used only for the D-point badge chip in `LeadCard` and the urgency indicator in the contextual suggestion card (`LeadDetailPage`). It follows the same `rgba(r,g,b,alpha)` inline style pattern as `TarefaStatusBadge`.

| Urgency State | Condition | Background | Text Color | Border Color | Label suffix |

Note on light mode: The existing `src/index.css` overrides `--red-hi`, `--amber-hi`, and `rgba()` backgrounds for the light theme via the `.light` class rules. The badge implementation must use CSS variable tokens (not hardcoded hex) so these overrides apply automatically.

## Component Specifications

### 1. D-point Badge in LeadCard (CAD-01)

**Replaces:** The existing cadência badge block at lines 322–334 of `LeadCard.tsx`.

**Trigger:** `getNextCadenciaPoint(lead, leadInteracoes)` returns a non-null result AND `isStagnant === false`.

**Do not show:** When lead is in a terminal stage, when `isStagnant === true`, or when `getNextCadenciaPoint` returns `null`.

**Position:** Same row as the existing stagnant badge — top-right badge row inside the LeadCard header, after the stagnant badge (which has priority when shown). When both are null, the row collapses.

**Anatomy:**
```
[Send icon w-2.5 h-2.5] [D{N}] [· {suffix}]
```
- Container: `<button>` (not `<div>`) — clicking navigates to `/mensagens?leadId=&stage=&nome=&empresa=&telefone=`
- Classes: `flex items-center gap-0.5 rounded px-1 py-0.5 text-[10px] font-medium transition-opacity hover:opacity-80 border`
- Style: `{ background, color, borderColor }` from the D-point urgency palette above
- `onPointerDown={e => e.stopPropagation()}` — mandatory DnD guard
- `onClick={e => { e.stopPropagation(); navigate(msgUrl) }}` — mandatory DnD guard

**Label format:**
- `daysUntil <= 0`: `"D{N} · hoje"` (chip is red)
- `daysUntil === 1`: `"D{N} · amanhã"` (chip is amber)
- `daysUntil >= 2`: `"D{N} · {N}d"` (chip is green) — example: `"D3 · 2d"`

**Title (tooltip):** `"{point.label} — {point.descricao}"` — e.g. `"Dia 3 — Envia follow-up de interesse; confirma dor principal"`

**Navigation URL:** `/mensagens?` with `URLSearchParams({ leadId, nome, empresa, stage: point.stage, telefone })`. Use `encodeURIComponent` for `telefone` before passing to URLSearchParams to avoid URL corruption (see RESEARCH.md Pitfall 3).

### 2. WhatsApp Navigation Button in LeadCard (WA-01)

**Position:** Same area as the existing message button at line ~381 of `LeadCard.tsx`. Placed adjacent to or replacing the current message shortcut, depending on existing button density.

**Design:** Icon-only button (no text label — space constraint in kanban card).

```
[MessageCircle icon w-3.5 h-3.5]
```

- Container: `<button>` with `onPointerDown={e => e.stopPropagation()}` and `onClick={e => { e.stopPropagation(); navigate(msgUrl) }}`
- Classes: `p-1 rounded hover:bg-[var(--alpha-bg-sm)] transition-colors`
- Style: `{ color: '#25D366' }` — WhatsApp green, always
- `title`: `"Abrir mensagem WhatsApp"` — required for accessibility
- Hidden when lead is in terminal stage (same guard as D-point badge)

**Only shown when `lead.telefone` is truthy.** If `lead.telefone` is empty/null, the button is not rendered.

**Navigation URL:** Same as the D-point badge navigation (identical URL) — both lead to MensagensPage with context pre-filled.

**Design:**

- Classes: `shrink-0 inline-flex items-center gap-1.5 h-8 px-3 rounded-md text-xs font-medium text-white`

**Placement:** Above the tab row (`"Informações"` / `"Histórico"` / etc.) in `LeadDetailPage`. Inserted as the first element in the main content area after the lead header.

**Condition:** Shown only when `getNextCadenciaPoint(lead, leadInteracoes)` returns non-null.

**Layout (flex row, `flex items-start gap-3 p-4 rounded-xl mb-4`):**

```
[Send icon] [Text column flex-1] [WhatsApp button]
```

**Icon:** `Send w-4 h-4 shrink-0 mt-0.5` at `color: #4ade80`.

**Text column:**
- Line 1 (title): `text-xs font-semibold` at `color: #4ade80` — format: `"{point.label} da cadência — {timing}"` where timing is:
  - `daysUntil === 0`: `"hoje"`
  - `daysUntil === 1`: `"amanhã"`

- Line 2 (body): `text-sm text-foreground mt-0.5` — `{point.descricao}` verbatim from `CADENCIA_DIAS`

**Container style:** `background: rgba(37,211,102,0.08); border: 1px solid rgba(37,211,102,0.20); border-radius: 0.75rem (rounded-xl)`

### 6. wa.me Actions in MensagensPage (WA-03)

**Scope:** Adding a "Copiar link" button alongside the existing "Abrir no WhatsApp" button.

**Position:** Immediately to the LEFT of the existing "Abrir no WhatsApp" `<a>` element (lines ~1092–1102). The two buttons form a horizontal button group.

**Button group container:** `flex items-center gap-2` wrapping both buttons.

**"Copiar link" button anatomy:**

```
[Copy icon w-3.5 h-3.5] Copiar link
```

Copied state (2s):

```
[Check icon w-3.5 h-3.5] Link copiado!
```

- Classes: `inline-flex items-center gap-1.5 h-8 px-3 rounded-md text-xs font-medium border transition-colors`
- Style: from the "Copiar link" color table in the Color section above
- `onClick`: calls `copyWaLink()` — `async function copyWaLink() { await navigator.clipboard.writeText(buildWhatsAppUrl(telefone, effectiveBody)); setWaLinkCopied(true); setTimeout(() => setWaLinkCopied(false), 2000) }`
- `title`: `"Copiar link wa.me para usar em outro dispositivo"`
- Guard: only rendered when `channel === 'whatsapp' && telefone` — identical guard as "Abrir no WhatsApp" (see RESEARCH.md Pitfall 5)

**Clipboard error handling:** Wrap `navigator.clipboard.writeText()` in `try/catch`. On failure: `toast.error('Não foi possível copiar o link. Copie manualmente o endereço.')`. This matches RESEARCH.md security note on clipboard API.

## Interaction Contracts

### D-point Badge — click behavior
- Single click navigates to `/mensagens` with full URL params (leadId, nome, empresa, stage, telefone)
- `onPointerDown` stops propagation before `@dnd-kit` captures the event
- `onClick` stops propagation and then navigates
- No hover tooltip delay — `title` attribute provides instant browser tooltip

### WhatsApp Button in LeadCard — click behavior
- Identical to D-point badge click behavior (same navigation URL)
- Both the D-point badge and the WA button navigate to the same URL — the stage param is derived from `getNextCadenciaPoint().point.stage`
- If both are rendered simultaneously, they are separate buttons (not merged) — the badge shows urgency, the WA button clarifies intent

### Contextual Suggestion Card — visibility
- Rendered whenever `getNextCadenciaPoint(lead, interacoes)` is non-null
- Not dismissable by the user (no close button) — it is informational, not an alert
- Updates reactively when `useInteracoesByLead(id)` data refreshes (TanStack Query automatic)

### "Copiar link" — copied state
- `waLinkCopied` state: `false` → `true` on click → `false` after 2000ms via `setTimeout`
- Icon swaps from `Copy` to `Check`; text swaps from `"Copiar link"` to `"Link copiado!"`
- Button remains clickable during the 2s window (does not disable — re-clicking resets the timer)
- No animation beyond the CSS `transition-colors` already declared on the button

### CadenciaPage — abordar() param addition
- The existing `abordar()` function adds `telefone` as a new URLSearchParam
- Behavior of the "Abordar" button is otherwise unchanged
- If `lead.telefone` is null/empty, `telefone` param is passed as empty string — MensagensPage already handles this (no wa.me button shown when telefone is empty)

## Accessibility Contracts

| Requirement | Implementation |
|-------------|----------------|
| D-point badge (interactive button without visible icon label) | `title="{point.label} — {point.descricao}"` on the `<button>` |
| WA button in LeadCard (icon only) | `title="Abrir mensagem WhatsApp"` on the `<button>` |
| WA button in LeadDetailPage | Visible text label "WhatsApp" + `title` attribute |
| Contextual suggestion card | Not a button — static `<div>` with color + text. Color is never the sole indicator of meaning (text label always present) |
| "Copiar link" button | Visible text label "Copiar link" / "Link copiado!" — no icon-only state |
| Copied state transition | Text changes within the button — screen readers re-announce the button label automatically |
| Color-only urgency (D-point chip) | Color reinforces urgency but the label text (`"hoje"` / `"amanhã"` / `"{N}d"`) conveys the same information without color |
