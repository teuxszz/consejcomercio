## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage                                              |
|-------|-------|----------------------------------------------------|
| xs    | 4px   | Icon-to-text gap, inline padding (`gap-1`)         |
| sm    | 8px   | Compact element spacing (`gap-2`, `px-2`, `py-2`)  |
| md    | 16px  | Default element spacing (`px-4`, `gap-4`)          |
| lg    | 24px  | Section padding (`p-6` = 24px, `gap-6`)            |
| xl    | 32px  | Layout gaps between major sections (`mb-8`)        |
| 2xl   | 48px  | Page-level vertical spacing                        |
| 3xl   | 64px  | Empty state padding (`py-12` = 48px used in project) |

Exceptions:
- Sidebar nav items: `px-3 py-2` (12px / 8px) ŌĆö existing pattern, do not change
- Badge pill: `px-1.5 py-0.5` (6px / 2px) ŌĆö Excecao tipografica formal: badge de contagem usa `text-[10px]`; valores sub-4px (`px-1.5 py-0.5`) sao necessarios para proporcao visual correta em texto de 10px; aprovados exclusivamente para badges de contagem numerica.
- Task row: `px-4 py-3` (16px / 12px) ŌĆö matches `TarefasPanel` existing pattern

## Color

### Semantic Tokens (from `src/index.css` ŌĆö use these, never hardcode hex)

| Role             | CSS Token                   | Dark value approx. | Light value approx. | Usage                                    |
|------------------|-----------------------------|--------------------|---------------------|------------------------------------------|

| Secondary (30%)  | `hsl(var(--card))`          | `#101e33`          | `#ffffff`           | Cards, modals, panels                    |

Never use accent or `cyan-500` for: borders, backgrounds of non-interactive surfaces, text on white/light.

## Component Specifications

### 1. TarefaModal (create / edit)

**Trigger:** "Nova Tarefa" button in `TarefasPage` or `LeadDetailPage`.

**Dimensions:** `max-w-lg` ŌĆö matches `NovaTarefaModal` in `TarefasPanel`.

**Mode detection:** Prop `mode: 'create' | 'edit'`. Title changes: "Nova tarefa" / "Editar tarefa".

**Fields (ordered top to bottom):**

| Field           | Element             | Validation                                        | Notes                                          |
|-----------------|---------------------|---------------------------------------------------|------------------------------------------------|
| Titulo          | `Input` autoFocus   | `z.string().min(3, 'Titulo obrigatorio (min. 3 caracteres)')` | Placeholder: "O que precisa ser feito?" |
| Descricao       | `Textarea` rows=3   | optional                                          | Placeholder: "Contexto, links, anotacoes..."   |
| Tipo            | `Select` (3-col grid) | enum (8 values)                                 | Default: `generica`                            |
| Prioridade      | `Select` (3-col grid) | enum (4 values)                                 | Default: `media`                               |
| Vencimento      | `Input type="datetime-local"` (3-col grid) | optional                       | Label: "Vencimento"                            |
| Responsavel     | `Select` populated from `usePerfis()` | `z.string().uuid('Responsavel obrigatorio')` | Shows user name; defaults to logged user |
| Notas           | `Textarea` rows=2   | optional, nullable                                | Show only in edit mode or as optional expand   |

**3-column grid:** Tipo / Prioridade / Vencimento ŌĆö `grid grid-cols-3 gap-3`.

**Lead context (create mode only):** When opened from `LeadDetailPage`, show read-only chip:
`"Lead: {lead.nome} ┬Ę {lead.empresa}"` ŌĆö `text-xs text-fg4 px-2 py-1 rounded bg-[var(--alpha-bg-sm)]` above the fields.

**Footer:**
- Left: `Button variant="outline"` ŌĆö "Descartar tarefa" (create mode) / "Descartar alteracoes" (edit mode)
- Right: `Button style={{ backgroundColor: '#0089ac' }}` ŌĆö "Criar tarefa" (create) / "Salvar alteracoes" (edit)
- Disabled state: submit button disabled while `isPending` or `!titulo.trim()`
- Loading state: submit button shows spinner icon + "Salvando..." text

**Form layout:** `<div className="space-y-3 py-2">` ŌĆö matches existing `NovaTarefaModal`.

**Labels:** `<label className="text-xs font-semibold text-fg2 mb-1 block">` ŌĆö updated from `font-medium` to `font-semibold` per typography contract (2 weights only: 400/600).

**Error display:** `<p className="text-xs text-red-500 mt-1">{errors.campo?.message}</p>` ŌĆö matches `NewLeadModal`.

## Interaction Contracts

### TarefaModal ŌĆö open/close behavior
- Opens: `Dialog open={open} onOpenChange={...}` ŌĆö Radix Dialog handles focus trap and ESC close
- Closes: ESC, Cancel button ("Descartar tarefa" / "Descartar alteracoes"), or backdrop click (when not submitting)
- Does NOT close while `isPending === true`
- On successful create: close modal + `toast.success('Tarefa criada')` + `qc.invalidateQueries`
- On error: keep modal open + `toast.error('Erro ao criar tarefa. Tente novamente.')`

### Complete task interaction
- Single click on `Circle` icon ŌĆö `useConcluirTarefa.mutate(id)`
- Optimistic: row visually fades out (opacity reduces to 0.5) immediately
- On success: row removed from list (query invalidated), badge count decrements
- On error: row restores, `toast.error('Erro ao concluir tarefa. Tente novamente.')`

### Realtime badge update
- Badge count is read-only rendered state ŌĆö never manually incremented/decremented in UI
- Source of truth is always the Realtime subscription + query re-fetch
- No optimistic update on badge ŌĆö it trails slightly (acceptable, matches Supabase Realtime latency)

### Pagination
