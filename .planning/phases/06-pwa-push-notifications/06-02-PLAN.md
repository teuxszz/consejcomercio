<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser SW → external URL (notificationclick → openWindow) | SW pode abrir URL arbitrária se deepLink for atacker-controlled |
| CDN/Vercel → browser (sw.js) | SW cached por longo período = bugs propagam lentamente |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-07 | Tampering | notificationclick → openWindow (open redirect) | mitigate | SW valida `new URL(targetUrl).origin === self.location.origin` antes de openWindow — Task 2.2 implementa explicit guard. R-S7 do RESEARCH §Security cobre. |
| T-06-08 | Information Disclosure | manifest.json expõe app structure pública | accept | Manifest é público por design (PWA spec). Não contém secrets. Risco aceitável. |
| T-06-09 | Tampering | SW cached agressivamente impede deploy de fix de segurança | mitigate | vercel.json header `Cache-Control: public, max-age=0, must-revalidate` para `/sw.js` (R-L4) — Task 2.2 implementa. |
| T-06-10 | Spoofing | Push notification body contém HTML/script malicioso | accept (low risk) | `showNotification(title, { body })` API trata body como plaintext (R-S4). Não interpreta HTML. Plan 03 escapa input do servidor (defense in depth). |
</threat_model>
