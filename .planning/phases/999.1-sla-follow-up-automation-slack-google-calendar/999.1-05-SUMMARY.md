## Decisions Made
- Upsert in `useCapturarTokenGoogle` omits `.select()` entirely (not just avoids reading fields) — the table's RLS has no SELECT policy at all (per Plan 02), so a `.select()` on the upsert would return an empty/error result; "returning minimal" is the correct and only viable mode here.
- Added a `useRef` guard on top of the `?google_linked=1` check in the capture effect, beyond what the plan's prose literally specified, as defense-in-depth against a re-render firing the effect twice before `setSearchParams` commits the cleared URL (matters because the refresh token is a use-once secret — Rule 2, missing critical safeguard for a security-sensitive one-time capture).
- Expiry display uses plain `Date.prototype.toLocaleString('pt-BR', ...)` rather than importing `date-fns`/`ptBR` for a single formatted string — avoids an unnecessary import for this one usage.

## Deviations from Plan

None beyond the defense-in-depth `useRef` guard noted above (Rule 2 — auto-added missing critical safeguard around one-time-secret capture; not a structural change, no user permission needed).
