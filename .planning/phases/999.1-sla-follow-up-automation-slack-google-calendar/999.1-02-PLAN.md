<objective>
Construir o COFRE de credenciais do Google Calendar: a tabela `google_calendar_tokens` (RLS zero-leitura — nem o dono lê; escrita só da própria linha), os RPCs SECURITY DEFINER que expõem apenas status (booleano + expiração) e desconexão, e o helper de Edge Function `_shared/google-calendar.ts` que refresca o access_token e cria o evento de follow-up via REST (raw fetch, sem SDK).

Purpose: refresh token do Google é credencial de terceiro, de longa vida (D-01) — precisa ficar inacessível ao browser mesmo em sessão comprometida (T-999.1-01). Só o service_role (edge function do Plan 03) lê o token; o browser apenas escreve a própria linha logo após o linkIdentity (Plan 05) e consulta status via RPC.
Output: migration 042 (tabela + RLS + 2 RPCs), `_shared/google-calendar.ts` + teste unitário com fetch mockado, e teste RLS `tests/rls/google_calendar_tokens.test.ts` (prova a negativa de SELECT).
</objective>
