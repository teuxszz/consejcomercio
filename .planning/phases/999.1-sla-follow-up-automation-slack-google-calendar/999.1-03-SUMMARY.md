# Phase 999.1 Plan 03: Motor de dispatch do SLA (sla.ts + notify-sla-evento) Summary

Helper `sendNotificacaoSla` com CAS atômico como primeiro statement, DM de warning sempre-envia com fallback gracioso para Google Calendar, escalação fail-safe para a gerência, e a Edge Function `notify-sla-evento` que expõe tudo isso com auth fail-closed e validação de payload.

## Performance

- **Duration:** ~30 min
- **Tasks:** 2 (Task 1 TDD RED→GREEN, Task 2 auto)
- **Files created:** 3

## Accomplishments
- `_shared/sla.ts` exporta `sendNotificacaoSla(supabase, { evento, leadId })`: CAS (`UPDATE lead_sla SET warning_sent_at|escalated_at=now() WHERE ... IS NULL RETURNING`) é o literal primeiro statement, antes de qualquer chamada de rede — se a linha não foi reivindicada, retorna `{ ok:true, skipped:'already_sent' }` sem tocar Slack/Calendar (SLA-04).
- `warning`: SEMPRE chama `postDm` quando há `slack_user_id` cadastrado (D-03 — não consulta `preferencias_notif`). Em seguida chama `getValidAccessToken`; com token, cria o bloco via `createFollowupEvent` e persiste `calendar_event_id` em `lead_sla`; sem token, faz fallback só-Slack sem quebrar (D-01/D-05, SLA-05). Erros do Calendar são isolados em try/catch e nunca abortam a DM já enviada.
- `escalonamento`: sem `SLACK_GERENCIA_CHANNEL_ID` configurado → fail-safe explícito (`{ ok:true, skipped:'no_gerencia_channel' }`), loga o skip e NUNCA lança exceção (D-04, SLA-07). Com o canal configurado, posta via `chat.postMessage` (helper local `postToChannel`, retry exponencial 3x para 429/5xx — espelha `postToSlack` de `notify-renovacao`).
- Log best-effort em `notificacoes_envios` (`tipo: sla_warning|sla_escalonamento`, `canal: 'slack'`) — fire-and-forget dentro de try/catch, nunca inspeciona `error.code` (23505 incluso é engolido por construção; a idempotência real vive no CAS).
- `notify-sla-evento/index.ts` espelha `notify-aprovacoes-stale`: OPTIONS/CORS, guarda de método (405), auth fail-closed (`WEBHOOK_SLA_SECRET` ausente → 500; `Authorization` inválido → 401 via `constantTimeAuthCheck`), parse JSON (400 em falha), validação de payload (`evento` ∈ {warning, escalonamento}, `lead_id` casa regex UUID — 400 antes de qualquer escrita), delega para `sendNotificacaoSla` e devolve o shape do resultado.
