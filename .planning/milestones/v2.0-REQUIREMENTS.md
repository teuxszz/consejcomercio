## v1 Requirements — 23/23 shipped

### Security Fixes (SEC) — pré-requisito para outras features

- [x] **SEC-01**: Tabela `tarefas` tem RLS aberta (`authenticated_all`) — usuário do tipo `cliente` consegue ler tarefas internas. Migration 033 deve adicionar policy `is_interno()` antes de qualquer feature de tarefas ir para produção. *(Phase 1 Plan 01-01)*
- [x] **SEC-02**: Adicionar coluna `notificar boolean DEFAULT true` na tabela `tarefas` para que tarefas derivadas de cadência possam optar por não disparar Slack DM automaticamente. *(Phase 1 Plan 01-01)*

### Tasks (TASK) — Sistema de Tarefas

- [x] **TASK-01**: Usuário pode criar tarefa por lead, com data de vencimento, responsável e descrição curta. *(Phase 1 Plan 01-02)*
- [x] **TASK-02**: Usuário pode criar tarefa interna do time (sem lead associado), com data e responsável. *(Phase 1 Plan 01-02)*
- [x] **TASK-03**: Usuário vê inbox de todas as tarefas atribuídas a ele, ordenadas por data de vencimento, com filtros por status (aberta / concluída). *(Phase 1 Plan 01-02)*
- [x] **TASK-04**: Usuário pode concluir, editar e excluir qualquer tarefa que criou ou que foi atribuída a ele. Coordenador+ pode gerenciar qualquer tarefa. *(Phase 1 Plan 01-02)*

### Cadence (CAD) — Guia de Cadência

- [x] **CAD-01**: Card de lead no Kanban exibe badge visual com o D-point atual da cadência (D1, D3, D5, D7, D10) calculado a partir de `cadencia_iniciada_em`. *(Phase 2 Plan 02-01)*
- [x] **CAD-02**: Existe uma vista "Ação Hoje" (CadenciaInbox) que lista apenas os leads cujo D-point cai no dia atual — o consultor abre o CRM e sabe exatamente com quem falar. *(Phase 2 Plan 02-01)*
- [x] **CAD-03**: Na página do lead e no CadenciaInbox, o sistema exibe sugestão de próxima ação contextual baseada no D-point. *(Phase 2 Plan 02-01)*
- [x] **CAD-04**: Sistema registra `cadencia_iniciada_em` no lead no momento em que o consultor marca o início da cadência. *(Phase 2 Plan 02-01)*

### WhatsApp Quick Actions (WA)

- [x] **WA-01**: Card de lead no Kanban e página do lead exibem botão WhatsApp que abre `wa.me/55{numero}?text={mensagem_encoded}` com pré-texto da cadência, sem custo de API. *(Phase 2 Plan 02-01)*
- [x] **WA-02**: O pré-texto do botão WhatsApp varia por D-point do lead. *(Phase 2 Plan 02-01)*
- [x] **WA-03**: Na página do lead, usuário pode copiar o link wa.me para a área de transferência com um clique. *(Phase 2 Plan 02-01)*

### Notifications (NOTIF) — Notificações

- [x] **NOTIF-01**: Usuário recebe DM no Slack quando uma tarefa é atribuída a ele por outra pessoa. *(Phase 3 Plan 03-01)*
- [x] **NOTIF-02**: Usuário recebe DM no Slack todo dia pela manhã com resumo: "Você tem X tarefas vencendo hoje e Y leads para contato na cadência" (pg_cron daily job — migration 034). *(Phase 3 Plan 03-02)*
- [x] **NOTIF-03**: Usuário recebe DM no Slack quando um lead dele chega no dia de ação da cadência (D1/D3/D5/D7/D10). *(Phase 3 Plan 03-02 — consolidado no mesmo cron diário)*
- [x] **NOTIF-04**: Sidebar do CRM exibe badge numérico com a contagem de tarefas abertas atribuídas ao usuário logado (Supabase Realtime subscription). *(Phase 1 Plan 01-03)*

### Adoption (ADOPT) — Visibilidade de Adoção

- [x] **ADOPT-01**: DashboardPage exibe card de adoção com logins únicos nos últimos 7 dias por usuário, leads criados na semana, tarefas criadas na semana. Visível para coordenador+. *(Phase 1 Plan 01-04 — AdocaoCard embedado no DashboardPage)*
- [x] **ADOPT-02**: Página `/adocao` (gated: coordenador+) exibe histórico de atividade por usuário: último login, leads registrados no mês, tarefas criadas, leads atualizados. *(Phase 1 Plan 01-04)*
- [x] **ADOPT-03**: DashboardPage e `/adocao` listam "leads esquecidos" — leads ativos sem nenhuma atualização há 7+ dias, com link direto para o lead. *(Phase 1 Plan 01-04 — LeadsEsquecidosTable)*

### Revenue (REV) — Dashboard de Receita

- [x] **REV-01**: Página `/receita` (gated: coordenador+) exibe MRR de contratos assessoria, ARR projetado, forecast dos próximos 3 meses, renovações vencendo em 30/60/90 dias. Separação por tipo de serviço. *(Phase 4 Plans 04-01 helpers + 04-03 UI)*
- [x] **REV-02**: Gráfico de evolução de MRR mês a mês (últimos 6 meses) usando `recharts` e `PeriodSelector` existentes. *(Phase 4 Plan 04-03 — ComposedChart com Line histórico + Area pontilhada forecast)*
- [x] **REV-03**: Lista de renovações pendentes com alerta visual para contratos vencendo em ≤30 dias. *(Phase 4 Plan 04-03 — Badge variant="destructive" + AlertTriangle no bucket ≤30d)*

## v2 Requirements (deferred to Milestone 3+)

- Notificações por e-mail como alternativa ao Slack (para usuários sem Slack)
- PWA / notificações push nativas no mobile
- Integração Notion (linkar atas e docs com leads/clientes)
- Portal do cliente — expansão com upload de documentos e aprovação de propostas
- Relatório de performance individual exportável (PDF/CSV)

---

## Out of Scope (mantido para Milestone 3+ decidir)

- **WhatsApp Bot/API** — exige número dedicado + Meta Business API + infraestrutura. Zero custo é incompatível com WhatsApp Business API real.
- **Integração de e-mail (inbox por lead)** — complexidade alta (OAuth Gmail/Outlook, threading, storage) para ROI incerto no curto prazo.
- **Multi-empresa / white-label** — CONSEJ only; sem plano de SaaS por ora.
- **Expansão do Portal do cliente** — fora do foco de adoção interna deste milestone.
- **IA para classificação automática de leads** — sem dados suficientes para treinar; retornar após 3 meses de uso real.
